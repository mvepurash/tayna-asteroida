# ТРЕБОВАНИЯ ЯНДЕКС ИГРЫ — НЕПРИЛОЖНЫЕ

## 1. SDK ИНИЦИАЛИЗАЦИЯ
- Обязательна инициализация YaGames SDK при загрузке страницы
- window.ysdk должен содержать ссылку на инициализированный объект
- При отсутствии SDK игра должна работать в режиме fallback (для локального тестирования)

## 2. ЖИЗНИ И РЕКЛАМА
- Базовая сессия: 3 жизни (CONFIG.CREW_SIZE = 3)
- ЧЕТВЁРТАЯ ЖИЗНЬ получается ТОЛЬКО после просмотра rewarded видео-рекламы
- Одна реклама на жизнь - не стакуется
- Минимальное расстояние между двумя видео-рекламами: 30 секунд (CONFIG.REWARDED_AD_COOLDOWN)

## 3. ЭКРАН МЕНЮ (ТИТУЛЬНЫЙ ЭКРАН)
- Игра ДОЛЖНА стартовать с меню, не с игрового процесса
- На меню две кнопки:
  * "НАЧАТЬ МИССИЮ" — запуск игры (спавн астронавта)
  * "НАСТРОЙКИ" — открыть меню настроек (звук, яркость и т.д.)
- Титульный экран: assets/title_screen.png (480x854)

## 4. ЭКРАН GAME OVER
- При смерти последней жизни показать экран Game Over
- На экране Game Over:
  * Отобразить финальную статистику (добычено кристаллов, время, рекорд)
  * Кнопка "СМОТРЕТЬ РЕКЛАМУ ДЛЯ +1 ЖИЗНИ" (если ads доступна, cooldown не истёк)
  * Кнопка "В МЕНЮ" — возврат на титульный экран
- Если кнопка рекламы недоступна (cooldown, нет связи) - показать это явно

## 5. ПАУЗА И РЕКЛАМА
- При показе любой рекламы игра ДОЛЖНА быть на паузе (Game.pause())
- После закрытия рекламы (успешно или пропуск) игра возобновляется (Game.resume())
- Кислород НЕ расходуется во время паузы (реализовано в Oxygen.pause())

## 6. LOADINGAPI
- При загрузке спрайтов показать прогресс-бар или "Loading..."
- После окончания загрузки вызвать ysdk.features.LoadingAPI.ready() (если ysdk доступен)
- Это уведомляет Яндекс о готовности игры

## 7. РАЗМЕР И ПРОИЗВОДИТЕЛЬНОСТЬ
- Сборка <10МБ (текущая: ~3.9МБ — OK)
- FPS ≥30 на мобильных устройствах
- Поддержка touch-событий (реализовано в input.js)
- Отсутствие memory leaks при долгой игре

## 8. СОХРАНЕНИЕ
- Локальное сохранение в localStorage работает (CONFIG.SAVE_KEY = 'tayna_asteroida_save')
- При возврате в меню → при входе в игру восстанавливается прогресс (опционально)
- При смерти и выборе рекламы: сохранять состояние для корректного продолжения

## 9. НАВИГАЦИЯ И ЖИЗНЕННЫЙ ЦИКЛ
- Start: Меню → кнопка "НАЧАТЬ МИССИЮ"
- Game: Спавн астронавта → геймплей
- Death (1-3): Респавн, жизни --
- Death (последняя без рекламы): Game Over экран
- Resume via Ad: +1 жизнь, продолжение геймплея
- Menu: Кнопка "В МЕНЮ" на Game Over или пауза (если добавить)

## 10. БЕЗОПАСНОСТЬ И СОВМЕСТИМОСТЬ
- HTML5 canvas API (ES6 поддержка)
- HTTPS необходим для Яндекс Игр
- CSP headers: script-src должна разрешить YaGames SDK
- Без eval(), с минимизацией при возможности
